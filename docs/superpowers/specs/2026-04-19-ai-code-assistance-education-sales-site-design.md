# AI 코드 어시스턴스 교육 프로그램 판매 사이트 디자인 스펙

## 1) 프로젝트 목표
- 사이트 목적: 40대 이상 성인 대상 AI 코드 어시스턴스 교육 프로그램 판매
- 1차 전환 목표: 설명회/체험 신청(예약)
- 핵심 약속: 비전공자도 따라오는 맞춤형 코칭
- 핵심 불안 요소: "내가 과연 따라갈 수 있을까"

## 2) 타깃 사용자
- 주 타깃: 40대 이상 성인
- 특성:
  - 비전공자 또는 디지털/코딩 학습 재입문자 비율이 높음
  - 실무/생업과 병행하며 학습 시간 제약이 큼
  - 학습 난이도와 완주 가능성에 대한 심리적 장벽이 큼

## 3) 제품/운영 조건
- 수업 방식: 온라인 라이브 + 녹화 복습
- 운영 기간: 12주 심화 과정
- 신청 후 연결 방식: 신청 폼 제출 → 문자/이메일 안내
- 핵심 신뢰 요소: 전/후 성과 사례 중심 제시
- 브랜드 톤: 차분하고 신뢰 중심형

## 4) 정보 구조(원페이지 랜딩)
전환 집중형 원페이지로 구성하며 섹션 순서는 아래와 같다.

1. 히어로 섹션
   - 핵심 메시지: 비전공자도 따라오는 맞춤형 코칭
   - 주요 CTA: 설명회 신청하기
2. 불안 해소 섹션
   - "내가 따라갈 수 있을까"에 대한 구체적 해소 방식 제시
3. 프로그램 소개 섹션
   - 12주 심화 과정 구조, 라이브/복습 방식 안내
4. 성과 사례 섹션
   - 전/후 비교 중심 결과 제시
5. 진행 방식 섹션
   - 신청부터 문자/이메일 안내까지 절차 설명
6. FAQ 섹션
   - 연령, 비전공 여부, 학습 난이도, 시간 투자 관련 질문 우선
7. 신청 폼 섹션(최종 CTA)
   - 최소 입력으로 완료 가능한 폼 구성

## 5) 컴포넌트 설계 원칙
- 톤앤매너: 과장/자극형 표현을 배제하고 근거 중심 문장 사용
- 카피 원칙:
  - 짧고 명확한 문장
  - 전문용어 최소화
  - "초보도 가능" 메시지는 반드시 근거와 함께 제시
- CTA 정책:
  - 상단/중단/하단 총 3회 반복
  - 버튼 문구는 "설명회 신청하기"로 통일
- 성과 카드:
  - Before/After 2열 비교
  - 수치 변화 또는 행동 변화 포함
- FAQ 정책:
  - 불안 해소형 질문 우선 배치
- 신청 폼 정책:
  - 이탈 방지를 위해 필수 항목 최소화

## 6) 데이터 흐름
1. 사용자가 CTA 클릭 후 신청 폼으로 이동
2. 필수 정보 입력 후 제출
3. 제출 성공 시 성공 메시지 노출
4. 성공 메시지 내 문자/이메일 안내 문구 제공

## 7) 예외 처리
- 필수값 누락: 필드 인라인 오류 메시지 표시
- 제출 실패: 재시도 버튼과 간단한 실패 안내 메시지 제공

## 8) 접근성/반응형 요구사항
- 접근성:
  - 큰 폰트 크기와 충분한 명도 대비 적용
  - 입력 라벨 명확화
  - 오류 텍스트를 시각적으로 분명하게 표시
- 반응형:
  - 모바일 우선 레이아웃
  - 큰 버튼 터치 영역
  - 짧은 문단 구성
  - 카드 컴포넌트 세로 정렬

## 9) 검증 기준
1. 상단/중단/하단 CTA 3개 모두 신청 폼으로 정상 이동
2. 필수값 검증이 정상 동작
3. 제출 성공/실패 메시지가 상황에 맞게 노출
4. 모바일 화면에서 가독성과 조작성이 유지
